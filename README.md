# Automated Ammunition and Equipment Inventory and Control System (AAE-ICS)

---

## Опис проєкту 

### Загальна інформація

Потрібно **2 студенти** для участі у **реальному прикладному проєкті для військової частини**. Це **не навчальний кейс**, а програмний продукт з реальним замовником, який після розробки буде використовуватися на практиці.

### Поточний стан проєкту

* проведені перші зустрічі з замовниками;
* зібрані початкові вимоги;
* отримані шаблони вхідних та вихідних документів, що використовуються у реальній роботі;
* ці документи є основою для проєктування структури БД та бізнес-логіки;
* **проєкт готовий до старту вже зараз**.

### Мета проєкту

Розробити **desktop-застосунок для обліку боєприпасів**, що автоматизує процеси надходження, видачі та переміщення ресурсів. Застосунок працює **офлайн**, з локальним сховищем даних.

### Основні завдання

1. Облік надходження, видачі та переміщення боєприпасів.
2. Автоматичне формування звітної документації (Excel, PDF).
3. Робота з друкованими документами (OCR, елементи AI).
4. Desktop-клієнт (опційно — мобільний для допоміжних задач).

### Технологічні вимоги (орієнтовно)

* Desktop: WPF або інша .NET desktop-технологія
* Платформа: .NET
* База даних: реляційна
* Зберігання: локальне
* Документи: Excel, PDF

### Організація роботи над проєктом

* робота ведеться **ітеративно**, з поступовим додаванням функціоналу;
* кожна ітерація завершується робочим результатом і тестуванням;
* проєкт **можна розпочати негайно**, без очікування формальних етапів.

### Формат участі та навчальні активності

Участь у проєкті може бути зарахована як:

* **виробнича / переддипломна практика**;
* **курсова робота**;
* основа для **бакалаврської або магістерської кваліфікаційної роботи**.

Проєкт розрахований на довгострокову участь і може розвиватися протягом кількох семестрів.

### Особливості проєкту

* реальний замовник і реальні дані;
* робота з первинною документацією;
* відповідальність за якість і стабільність рішення;
* досвід повного циклу розробки — від аналізу до впровадження.

### Перспективи розвитку

* масштабування функціоналу;
* створення веб-клієнта;
* адаптація під інші сфери обліку та інвентаризації.
Просто збережи інформацію
